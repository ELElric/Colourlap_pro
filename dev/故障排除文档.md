# ColorLab Pro — 故障排除与问题分析文档

> 更新时间：2026-06-20 20:30

---

## 1. 问题汇总

### 1.0 CSV 导入后 R 光谱显示异常（最新）

**现象**：导入 CSV 文件后，R 光谱在表格中显示为占位符，无数据且无法预览。G、B 光谱正常。

**数据验证**：通过命令行测试确认数据完整：
- R 光谱：401 个波长点，83 个非零值，峰值 630nm（值=1.0）
- G 光谱：401 个波长点，峰值 530nm
- B 光谱：401 个波长点，峰值 460nm

**可能原因**：
1. GUI 未刷新显示导入的数据
2. 表格选中状态问题
3. 图表渲染问题

**建议操作**：
1. 导入后点击表格中的 R 光谱行（不是复选框）
2. 检查光谱预览区域是否有数据
3. 如果仍无显示，尝试重启 GUI

### 1.1 导入时选择类别卡死

**现象**：点击 Import CSV/Excel 后，弹出类别选择对话框时 GUI 卡死无响应。

**原因**：`QInputDialog.getItem()` 是模态对话框，会阻塞 Qt 主线程事件循环。在信号处理器中调用模态对话框会导致 GUI 冻结。

**解决方案**：移除导入时的类别选择对话框。改为导入后通过右键菜单 → "Set Category" 设置类别。

**涉及文件**：
- `src/colorlab_pro/ui/pages/spectrum_page.py` — `_on_import_csv()`, `_on_import_xlsx()`

---

### 1.2 数据库无项目导致导入失败

**现象**：导入文件时报错 "Failed to import spectrum: No project selected."

**原因**：移除 `load_default_spectra()` 后，启动时不再创建默认项目。`current_project_id` 为 None，导入时无项目可写入。

**解决方案**：在 `app.py` 中启动时自动创建空的默认项目：
```python
if main_ctrl.current_project_id is None:
    project_ctrl = ProjectController(main_ctrl)
    pid = project_ctrl.create_project("Default Project")
    if pid is not None:
        main_ctrl.set_current_project(pid)
```

**涉及文件**：
- `src/colorlab_pro/ui/app.py` — `main()`

---

### 1.3 通道检测错误

**现象**：`R110B-2` 被检测为 B 通道，`G130B-6` 被检测为 B 通道。

**原因**：通道检测逻辑基于材料名称，但名称中的 RGB 后缀不代表通道。通道应根据光谱数据的实际透过率峰值来判断。

**解决方案**：
1. 导入时使用光谱数据检测通道（`detect_channel(spectrum)`）
2. 不再依赖材料名称中的 RGB 后缀
3. R: 600-700nm, G: 500-580nm, B: 400-500nm

**涉及文件**：
- `src/colorlab_pro/engines/spectrum_normalizer.py` — `detect_channel()`
- `src/colorlab_pro/services/spectrum_service.py` — `import_spectrum()`

---

### 1.4 光谱选择器显示错误数据

**现象**：Gamut Calculator 页面的 RGB 选择器显示 CF 彩膜光谱，透射率选择器显示 LED 发光光谱。

**原因**：`refresh_spectrum_list()` 没有过滤类别，所有光谱都显示在所有选择器中。

**解决方案**：
- RGB 选择器：仅显示 `category=LED` 的光谱
- 透射率选择器：仅显示 `category=CF` 的光谱
- White 选择器：显示所有光谱

**涉及文件**：
- `src/colorlab_pro/ui/pages/gamut_calculator_page.py` — `refresh_spectrum_list()`
- `src/colorlab_pro/ui/pages/thickness_optimizer_page.py` — `refresh_spectrum_list()`

---

### 1.5 导航栏拖拽排序后工作区不跟随

**现象**：拖拽导航栏项目调整顺序后，点击导航项显示的页面不对。

**原因**：
1. `app.py` 中 `currentRowChanged` 信号连接到 `main_ctrl.switch_to_page`，绕过了正确的导航逻辑
2. `_stack_names` 使用空字符串作为页面名称（因为 `windowTitle()` 返回空）

**解决方案**：
1. 移除 `app.py` 中的 `currentRowChanged.connect(main_ctrl.switch_to_page)`
2. 在 `add_page()` 中显式传入页面名称
3. 使用 `_sidebar_order` 和 `_stack_names` 映射导航

**涉及文件**：
- `src/colorlab_pro/ui/main_window.py` — `MainWindow`
- `src/colorlab_pro/ui/app.py` — `main()`

---

### 1.6 Settings 页面问题

**现象**：Settings 占用一个工作区页面，用户不需要。

**原因**：Settings 被设计为独立页面，但实际只需要一个对话框。

**解决方案**：
1. 从导航栏移除 Settings
2. 添加到菜单栏：`Settings (Ctrl+,)`
3. 点击弹出设置对话框

**涉及文件**：
- `src/colorlab_pro/ui/main_window.py` — `_build_menu_bar()`, `_on_settings_clicked()`
- `src/colorlab_pro/ui/app.py` — 移除 SettingsPage

---

### 1.7 自动刷新未连接

**现象**：切换页面后下拉框为空，需要手动刷新。

**原因**：`app.py` 中 `connect_auto_refresh` 循环遍历元组列表时，迭代的是 `(name, page)` 元组而不是页面对象。

**解决方案**：
```python
# 错误：
for _i, page in enumerate(pages):

# 正确：
for _name, page in pages:
```

**涉及文件**：
- `src/colorlab_pro/ui/app.py` — `main()`

---

### 1.8 类别未正确传递

**现象**：导入时选择的类别未应用到光谱数据。

**原因**：
1. `import_spectrum()` 中从 meta 读取类别后，又被 `category_from_channel()` 覆盖
2. 去重时返回旧数据的 ID，旧数据的类别未更新

**解决方案**：
1. 从 meta 读取 `default_category` 优先于自动检测
2. 去重时更新旧数据的类别和通道

**涉及文件**：
- `src/colorlab_pro/services/spectrum_service.py` — `import_spectrum()`
- `src/colorlab_pro/repositories/spectrum_repository.py` — `find_duplicate()`, `update_spectrum_fields()`

## 3. 操作指南

### 3.1 导入数据

1. 点击 "Import CSV" 或 "Import Excel"
2. 选择文件 → 直接导入（不弹对话框）
3. 选中导入的光谱 → 右键 → "Set Category" → 选择 LED/CF/QD/白光

### 3.2 设置类别

1. 在光谱库中选中光谱
2. 右键 → "Set Category"
3. 选择 LED / CF / QD / 白光

### 3.3 设置通道

1. 在光谱库中选中光谱
2. 右键 → "Set Channel"
3. 选择 R / G / B / Auto-detect

### 3.4 多光谱对比

1. 在光谱库中勾选多个光谱的复选框
2. 光谱图自动叠加显示选中的光谱曲线
3. 不同通道的光谱使用不同颜色（R=红色, G=绿色, B=蓝色）
